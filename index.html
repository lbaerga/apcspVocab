Algorithm

Algorithm

A finite set of instructions that accomplish a specific task.
Every algorithm can be constructed using combinations of sequencing, selection, and iteration

Pseudocode

Pseudocode

Notation that looks like a simplified programming language, it's used in the program design stage.

Strong Password

Strong Password

A strong password is hard to guess but easy to remember.
Brute force hacking relies on dictionary words and easy-to-guess substitutions in PWs, like @ for the letter "a".

Selection

Selection

determines what part(s) of an algorithm are executed, based on a condition being true or false.

Sequencing

Sequencing

application of each step of an algorithm in the order in which the code statements are given.

Iteration

Iteration

is a repeating portion of an algorithm. Iteration repeats a specified number of times or until a given condition is met.

Encryption

Encryption

is the process of encoding data to prevent unauthorized access.

Decryption

Decryption

is the process of decoding data.

Brute Force Attack

Brute Force Attack

is when an attacker tries many passwords or passphrases, hoping that they will eventually guess correctly. All possible passwords and passphrases are systematically checked, until the correct one is found. Or, the attacker can attempt to guess the key.

Cybersecurity

Cybersecurity

measures taken to protect a computer or computer system (as on the internet) against unauthorized access or attack.

Decidable Problem

Decidable Problem

A decidable problem is a decision problem for which an algorithm can be written to produce a correct output for all inputs (e.g., “Is the number even?”).

Undecidable Problem

Undecidable Problem

An undecidable problem is one in which no algorithm can be constructed that always leads to a correct yes-or-no answer.

Scalability

Scalability

Scalability is the capacity for a system to change in size and scale to meet new demands.

Efficiency

Efficiency

Efficiency is an estimation of the amount of computational resources used by an algorithm. Efficiency is typically expressed as a function of the size of the input.
